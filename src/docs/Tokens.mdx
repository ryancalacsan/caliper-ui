# Design tokens

Tokens are the single source of truth for color, type, spacing, radii, motion,
and stacking order. They live as SCSS maps in `src/tokens/`. A build step walks
those maps once and emits a matching CSS custom property on `:root`, so a value
is written in exactly one place and reaches components two ways:

- `var(--color-primary)` for runtime theming (the common case in components)
- `fn.color('primary')` for compile-time work like `calc()` and media queries

Change a token in `src/tokens/`, and both paths update from the same line. No
component holds a raw hex or pixel value.

## Color

The palette is "Editorial Noir": mostly monochrome, with one expressive accent.
Primitives are raw ramps — **ink** (cool near-black), **bone** (warm paper
neutrals), **iris** (the blue-violet accent), plus danger and success. Components
never use a primitive. They use the semantic roles below, chosen so every
text-on-surface pairing clears WCAG 2.2 AA in both themes. The pairings are
verified in CI-adjacent fashion by `npm run test:contrast`.

| Role | Custom property | Light → Dark | Min contrast |
| --- | --- | --- | --- |
| Text | `--color-text` | ink 950 → bone | 17.2:1 |
| Muted text | `--color-text-muted` | ink 600 → ink 300 | 7.9:1 |
| Subtle text | `--color-text-subtle` | ink 500 → ink 400 | 4.9:1 |
| Primary fill | `--color-primary` | iris 600 → iris 400 | — |
| Label on primary | `--color-text-on-primary` | white → ink 950 | 5.8:1 |
| Accent text | `--color-accent-text` | iris 600 → iris 300 | 4.9:1 |
| Danger text | `--color-danger` | danger 700 → danger 400 | 6.2:1 |
| Danger button | `--color-danger-solid` | danger 700 → danger 600 | 4.8:1 |
| Field border | `--color-border-strong` | ink 400 | 3.1:1 |

Two distinctions the system makes deliberately:

- **`primary` (fill) vs `accent-text`.** On a dark surface a tint cannot be dark
  enough for the iris fill color to clear AA as text, so accent text and links
  use a lighter `accent-text` role while the button fill stays punchy.
- **`danger` (text) vs `danger-solid` (button).** Error text wants a light red on
  dark; a danger button wants a darker red so its white label passes. Same value
  in light, different in dark.

The signature accent is `--color-gradient-accent`, an iridescent blue-to-violet
gradient used sparingly (a hero word, a focus glow), never behind body text.

### Theming

The semantic roles are emitted twice, from two SCSS maps: `$semantic` for light
and `$semantic-dark` for dark. Components only ever read a role, so the theme
switches with no component changes. Use the theme switcher in the Storybook
toolbar to flip every story, and watch the Accessibility panel to confirm the
dark theme holds the same AA bar.

- `[data-theme="dark"]` on the root opts in to dark.
- `[data-theme="light"]` forces light, even under an OS dark setting.
- With no attribute set, `prefers-color-scheme` decides.

Both themes clear AA. One split the dark theme made explicit: `--color-danger`
is the text color (a light red on dark), while `--color-danger-solid` is the
danger button background (a darker red so its white label passes). They are the
same value in the light theme and diverge in the dark one.

## Typefaces

An editorial pairing, self-hosted with Fontsource so there are no third-party
requests and no font-swap flash.

| Family | Custom property | Used for |
| --- | --- | --- |
| Fraunces | `--font-display` | display headings, the serif voice |
| Hanken Grotesk | `--font-sans` | UI and body |
| Geist Mono | `--font-mono` | code, token names, uppercase micro-labels |

Headings adopt the serif at the base-element level, since in a design system the
type ramp is part of the foundation rather than something each page re-invents.

## Type scale

A 1.25 (major third) modular scale on a 1rem / 16px base, extended up top for
display work. Big display sizes against small letter-spaced labels is the
editorial move.

| Token | Custom property | Size |
| --- | --- | --- |
| xs | `--text-xs` | 0.75rem / 12px |
| sm | `--text-sm` | 0.875rem / 14px |
| base | `--text-base` | 1rem / 16px |
| md | `--text-md` | 1.125rem / 18px |
| lg | `--text-lg` | 1.25rem / 20px |
| xl | `--text-xl` | 1.5rem / 24px |
| 2xl | `--text-2xl` | 1.875rem / 30px |
| 3xl | `--text-3xl` | 2.25rem / 36px |
| 4xl | `--text-4xl` | 3rem / 48px |
| 5xl | `--text-5xl` | 4rem / 64px |
| 6xl | `--text-6xl` | 5.25rem / 84px |

Weights run `--weight-regular` 400 through `--weight-black` 900 (Fraunces is
variable, so the display face can push further). Line heights: `--leading-none` 1
for display through `--leading-relaxed` 1.75. Tracking ranges from
`--tracking-tightest` for big display to `--tracking-wider` for uppercase labels.

## Spacing

A 4px base grid. Every margin, padding, and gap snaps to one of these so density
and rhythm stay even across components.

| Token | Custom property | Value |
| --- | --- | --- |
| 2xs | `--space-2xs` | 0.25rem / 4px |
| xs | `--space-xs` | 0.5rem / 8px |
| sm | `--space-sm` | 0.75rem / 12px |
| md | `--space-md` | 1rem / 16px |
| lg | `--space-lg` | 1.5rem / 24px |
| xl | `--space-xl` | 2rem / 32px |

## Z-index

A named stacking scale. Order is intentional rather than a contest of magic
numbers.

| Token | Custom property | Value |
| --- | --- | --- |
| dropdown | `--z-dropdown` | 1000 |
| sticky | `--z-sticky` | 1100 |
| overlay | `--z-overlay` | 1200 |
| modal | `--z-modal` | 1300 |
| popover | `--z-popover` | 1400 |
| toast | `--z-toast` | 1500 |

## Motion

Durations (`--duration-fast` 120ms to `--duration-slow` 320ms) and easings
(`--ease-standard`, `--ease-emphasized`, `--ease-exit`). Every animated property
is wrapped in a `prefers-reduced-motion: no-preference` guard, so motion never
plays for users who asked the OS to reduce it.

## Breakpoints

Named viewport widths in rem, so they respond to the user's base font size. A
media query condition cannot read a custom property, so unlike the other tokens
these are consumed in Sass through the `media-up` / `media-down` mixins rather
than as `var(--...)`.

| Token | Value | Target |
| --- | --- | --- |
| sm | 30rem / 480px | large phone |
| md | 48rem / 768px | tablet |
| lg | 64rem / 1024px | small laptop |
| xl | 80rem / 1280px | desktop |

```scss
@include mix.media-down('md') {
  padding-inline: var(--space-lg);
}
```
