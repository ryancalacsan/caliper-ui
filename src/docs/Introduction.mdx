# Bespoke

A small, accessible React component library. Hand-written SCSS with BEM, design
tokens as the single source of truth, and accessibility as a first principle.
Eight components, two themes, both held to WCAG 2.2 AA.

## How this is organized

- **Foundations / Design tokens** — the color, type, spacing, radii, and motion
  tokens every component is built from. Start here.
- **Components** — Button, TextField, Modal, Checkbox, RadioGroup, Select, Tabs,
  and Tooltip. Each has prop controls, an Accessibility panel (axe), and a
  documented usage page.

## Try the system

- Use the **theme switcher** in the toolbar to flip every story between the
  light (Daylight) and dark (Noir) themes.
- Open the **Accessibility** panel on any story to see the axe checks run live.
- Open the **Interactions** panel on Button, Tabs, Select, Modal, Tooltip,
  Checkbox, or RadioGroup to watch the keyboard flows replay step by step.

## Principles

1. **Tokens are the source of truth.** No component holds a raw hex or pixel
   value. A theme is a second set of token values, nothing more.
2. **Accessibility is structural.** Real semantic elements, correct ARIA, managed
   focus, visible focus indicators, `prefers-reduced-motion` respected, and AA
   contrast verified by script.
3. **The CSS is the artifact.** Hand-written, structured SCSS with BEM. No utility
   framework, on purpose.
